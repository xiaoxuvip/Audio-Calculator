三、编译命令详解

3.1 前置条件

  1. 安装 .NET 8.0 SDK（下载地址：https://dotnet.microsoft.com/download/dotnet/8.0）
  2. 打开终端（PowerShell 或 CMD），输入 dotnet --version 确认安装成功
  3. 进入项目根目录（包含 AudioCalculator.csproj 的文件夹）
⚠️ 注意 以下编译命令都是「一次性复制整段」粘贴到终端执行的，不需要逐行输入。每段命令就是一条完整的 dotnet publish 命令。

3.2 编译轻量版（Lite）

轻量版体积较小（约 1-2 MB），但运行时需要用户电脑已安装 .NET 8.0 运行时。

PowerShell 版本

PowerShell — 编译轻量版（一次性复制执行）
dotnet publish AudioCalculator.csproj `
  -c Release `
  -r win-x64 `
  -p:PublishSingleFile=true `
  -p:SelfContained=false `
  -p:DefineConstants="EDITION_LITE" `
  -o ./publish-lite

CMD 版本

CMD — 编译轻量版（一次性复制执行）
dotnet publish AudioCalculator.csproj -c Release -r win-x64 -p:PublishSingleFile=true -p:SelfContained=false -p:DefineConstants="EDITION_LITE" -o ./publish-lite

3.3 编译完整版（Full）

完整版自带 .NET 运行时，体积较大（约 60-80 MB），但用户无需安装任何依赖即可运行。

PowerShell 版本

PowerShell — 编译完整版（一次性复制执行）
dotnet publish AudioCalculator.csproj `
  -c Release `
  -r win-x64 `
  -p:PublishSingleFile=true `
  -p:SelfContained=true `
  -p:IncludeNativeLibrariesForSelfExtract=true `
  -p:DefineConstants="EDITION_FULL" `
  -o ./publish-full

CMD 版本

CMD — 编译完整版（一次性复制执行）
dotnet publish AudioCalculator.csproj -c Release -r win-x64 -p:PublishSingleFile=true -p:SelfContained=true -p:IncludeNativeLibrariesForSelfExtract=true -p:DefineConstants="EDITION_FULL" -o ./publish-full

3.4 手把手执行步骤

  1. 打开 PowerShell 或 CMD 终端
  2. 进入项目目录：cd 你的路径\AudioCalculator
  3. 确认目录下有 AudioCalculator.csproj 文件
  4. 复制上面的编译命令（整段复制），粘贴到终端，按回车执行
  5. 等待编译完成，输出文件在 publish-lite/ 或 publish-full/ 目录中
  6. 找到 AudioCalculator.exe 即为最终产物

3.5 编译参数说明 & 版本对比

八、完整发版流程

  1. 修改版本号 — 按照第二节的检查清单，同步修改 2 个位置的版本号
  2. 测试功能 — 在 IDE 中运行调试，确认功能正常（拖拽文件、计费、截图、右键菜单）
  3. 编译轻量版 — 执行第三节的轻量版编译命令，检查 publish-lite/AudioCalculator.exe
  4. 编译完整版 — 执行第三节的完整版编译命令，检查 publish-full/AudioCalculator.exe
  5. 测试编译产物 — 双击运行编译出的 exe，确认标题栏版本号正确、功能正常
  6. 打包分发 — 将 exe 文件压缩为 zip，上传到分发渠道

九、常见问题 FAQ

Q1: 编译报错 "找不到 .NET SDK"

✅ 解决方案 确认已安装 .NET 8.0 SDK，运行 dotnet --version 检查。如果已安装但仍报错，尝试重启终端或检查 PATH 环境变量。

Q2: 右键菜单添加失败

✅ 解决方案 需要管理员权限。右键点击 exe → "以管理员身份运行"，然后再点击"添加右键"按钮。也可以直接双击 RegisterContextMenu.reg 注册表脚本。

Q3: 截图功能不工作

✅ 解决方案 截图功能使用 WPF 的 RenderTargetBitmap 渲染，不依赖外部库。如果剪贴板复制失败，可能是其他程序占用了剪贴板，关闭剪贴板管理工具后重试。

Q4: 音频时长与资源管理器显示不一致

✅ 解决方案 程序优先使用 Windows Shell API 获取时长（与资源管理器一致），如果 Shell API 失败才回退到 TagLib。如果仍有差异，可能是文件元数据损坏。

Q5: 轻量版和完整版有什么功能区别？

✅ 解答 功能完全一致，唯一区别是：轻量版不包含 .NET 运行时（体积小，需要用户已安装 .NET 8），完整版自带运行时（体积大，开箱即用）。标题栏会显示对应的版本类型。
